日期的处理

  • 数据库中的字段类型为 date
  • 实体类使用日期属性 @JsonFormat ，换成 JSON 时就会把日期按照这个格式转换。
  • 前端的 date 表单元素使用和第二步一样的 pattern，例如 yyyy-MM-dd，这样就可以在前端正常显示日期了。
@Data
@AllArgsConstructor
@NoArgsConstructor
public class Student {
    private Integer id;
    private String name;
    //@JsonProperty("address")
    private String addr;
    /*
    功能：
        1.将前端的JSON值按照pattern映射到对应的属性
        2.将服务器响应的数据按照pattern转换成JSON
    */
    @JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss", timezone = "GMT+8")
    /*
    功能：
        1.将前端传过来的日期字符串（如url-String格式、Form-Data格式，不能处理JSON）按照pattern转化为对应的Date类型
        2.不会影响服务器返回前端的时间类型数据格式
    */
    @DateTimeFormat(pattern = "yyyy-MM-dd HH:mm:ss")
    private Date birthday;

}

记住账户

实现思路：登入灯光时并且勾选记住账户时，服务器端生成一个特定的、加密的标识符发送给浏览器，存储在用户浏览器本地的 Cookie 中，同时服务器端也在数据库中存储这个 cookie，下次用户访问的时候携带此 cookie，服务器端将此 cookie 和数据库中的信息进行对比。

  1. 生成 Token：当用户成功登录并选择“记住我”选项时，服务器会生成一个唯一的、加密的 Token。这个 Token 可能包含用户的唯一标识（如用户 ID），并且可以加入时间戳和其他信息以增强安全性。

  2. 存储 Token：服务器将这个加密的 Token 发送给用户浏览器，存储在 Cookie 中。同时，服务器也会在数据库中保存这个 Token 的记录，以便将来验证。

  3. 设置有效期：这个 Cookie 的有效期通常会设置较长，比如几天或几周，这样即使用户关闭浏览器或计算机，Cookie 也不会立即失效。

  4. 用户再次访问：当用户在有效期内再次访问网站时，浏览器会自动将这个 Cookie（包含 Token）发送给服务器。

  5. 验证 Token：服务器接收到请求后，会读取 Cookie 中的 Token，并与数据库中存储的 Token 记录进行匹配验证。如果匹配成功，并且 Token 没有过期，服务器就可以识别用户的身份。

  6. 自动登录：一旦 Token 验证通过，服务器就可以为用户创建一个新的会话（Session），使用户无需再次输入账号密码即可访问网站。

  7. 安全措施：为了提高安全性，Token 通常会进行加密处理，以防止信息泄露。此外，还可以采取其他措施，如限制 Token 的使用范围（只在特定设备或 IP 地址上有效）、定期更新 Token 等。

cookie

如果只是生成一个 cookie 发给浏览器，是做不到区分验证用户的，除非你再设计一套验证机制。比如浏览器携带 cookie 再次访问时，服务器端解析这个 cookie，并且进行验证。

token

Token 在认证和授权过程中，通常是指一个包含安全凭证的字符串，这个凭证代表了用户的身份信息或会话信息。Token 并不限于只作为 Cookie 的值出现，它可以通过多种方式传输，例如 HTTP 头部、URL 参数等。Token 的核心作用是提供一种安全的方式来验证用户身份或会话有效性，而无需每次都传输用户名和密码。

Token 通常具有以下特点：

  1. 唯一性：每个 Token 都是唯一的，以确保每个用户或会话都可以被明确地识别。

  2. 有限的有效期：Token 通常具有过期机制，过期后无法再使用，以减少被盗用的风险。

  3. 包含信息：Token 内部可以包含一些基本信息，如用户的 ID、创建时间、过期时间等。这些信息可以通过 Token 自身的结构直接编码（如 JWT），也可以是加密的，需要特定的密钥才能解密。

  4. 加密：为了安全起见，Token 通常会进行加密处理，或者至少对关键信息进行加密，以防止信息被泄露。

因此，当我们提到“记住我”功能中的 Token 时，它不仅仅是 Cookie 值中的一个字段，更是一种包含用户身份验证信息的安全凭证。服务器可以通过解析和验证这个 Token ，或者对比数据库，来确认用户身份，从而实现自动登录或维持会话的目的。

session

使用 session 可以实现对用户身份的验证，当服务器端设置好 session 之后，我们不用在写什么特殊的验证机制就可以实现在单次回话中对用户身份的验证。所以不适用于来实现记住账户功能。

//判断cookie信息
Cookie[] cookies = req.getCookies();    //获取请求中携带的cookie信息
if (cookies != null) {
	for (Cookie cookie : cookies) { //遍历cookies数组，寻找loginToken
		if (cookie.getName().equals("loginToken")) {
			String loginToken = cookie.getValue();
			String decrypt = EncryptionUtil.decrypt(loginToken);    // 解密token
			String[] decryptParts = decrypt.split(":"); //切割解密后的字符串

			String username = decryptParts[0];  //用户名
			String timestamp = decryptParts[1]; //记住账户那一刻的时间
			if (new Date().getTime() - Long.parseLong(timestamp) < 432000000) {//若loginToken没有过期（若时间差小于5天）
				
				try {
					Admin admin = adminService.findByName(username);    //根据用户名查找到用户其他信息
					session.setAttribute("admin", admin);   //设置session
					RespBean ok = RespBean.ok("自动登录成功！");   //设置响应对象
					JsonUtil.toJSON(resp.getOutputStream(), ok);    //向前端返回JSON格式的响应信息
				} catch (SQLException e) {
					throw new RuntimeException(e);
				} catch (ThrowInfoException e) {
					throw new RuntimeException(e);
				}
			}
		}
	}
}
if (rememberMeInFiveDays.equals("1")) { //如果用户选择了记住我  
    // 创建一个Cookie，并且设置加密信息（用户名+时间戳）  
    String encryptedToken = EncryptionUtil.encrypt(username, new Date().getTime());  
    Cookie loginToken = new Cookie("loginToken", encryptedToken);  
    // 设置Cookie的有效期为5天  
    loginToken.setMaxAge(5 * 24 * 60 * 60);  
    // 将Cookie设置为HttpOnly，防止XSS  
    loginToken.setHttpOnly(true);  
    // 将Cookie发送给客户端  
    resp.addCookie(loginToken);
