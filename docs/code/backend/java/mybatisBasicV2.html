使用别名

相当于扫描 pojo 包下面所有实体类，配置了一个别名，例如 pojo 下有 User Brand 类，在 ResultType 中就可以直接使用 user/User，而不必再写全类名。

<typeAliases>
	<package name="com.blueboy.pojo"/>
</typeAliases>

注：也可以单独给类配置别名，另外 Mybatis 自动给基本类型配置了别名，之后可以用 string/String 即可用到这个类型。

使用代理开发(重点)

需要把 UserMapper.xml 和 UserMapper 接口 放在同一个包下。将在 resources 下的 UserMapper.xml 与接口具有相同的层级结构就可以最终编译放在一个文件夹中。这样接口和 xml 配置文件才可以互相绑定。 需要遵循的规范：

  • Mapper.xml文件中的namespace与mapper接口的全限定名相同；
  • Mapper接口方法名和Mapper.xml中定义的每个statement的id相同；
  • Mapper接口方法的输入参数类型和mapper.xml中定义的每个sql的parameterType的类型相同；
  • Mapper接口方法的返回值类型和mapper.xml中定义的每个sql的resultType的类型相同。
    • Mapper 接口方法返回值类型时 List<xxx>时，resultType 写 xxx 方法三要素 + 层次结构

MybatisDemo2.java

package com.blueboy.test;  
  
public class MybatisDemo2 {  
    public static void main(String[] args) throws IOException {  
        /*加载文件，创建sqlSessionFactory对象*/  
        String resource = "mybatis-config.xml";  
        InputStream inputStream = Resources.getResourceAsStream(resource);  
        SqlSessionFactory sqlSessionFactory = new SqlSessionFactoryBuilder().build(inputStream);  
  
        /*获取sqlSession对象，用这个执行sql*/  
        SqlSession sqlSession = sqlSessionFactory.openSession();  
  
        //执行sql方式1  
        //List<User> userList = sqlSession.selectList("test.selectAll");  
        
        //使用Mapper代理执行sql  
        /*        
        * 创建出一个 UserMapper接口中的方法 与 UserMapper.xml 中 id 关联 的接口实现类的对象  
        * 类比：之前使用传统JavaWeb开发时，我们需要写Dao接口和DaoImpl，
        * 用的时候使用 Impl实现对象进行调用方法实现增删改查。
        * 现在使用代理，这些操作都被框架给代理了，当于框架给我们做了创建有那些statament语句的Impl。
        * 并且给了一个Impl类的实现对象。
        * 通过调用mapper对象中的方法，实现操作  
        * */     
    
        //获取UserMapper接口实现类的对象
        UserMapper mapper = sqlSession.getMapper(UserMapper.class);  
        List<User> userList = mapper.selectAll();  
          
        System.out.println(userList);  
  
        //释放资源  
        sqlSession.close();  
    }  
}

使用包扫描的时候流程也是如上，只不过之前时一个一个 xml 找接口再绑定，现在是批量 xml。批量找到对应的接口然后绑定，最后可以直接通过 getMapper 用了。

编写映射文件

数据库字段和属性不匹配问题

如果存在这种问题，推荐使用 resultMap 方式，之后在 statement 中不再写 resultType，而写 resultMap，实现了映射字段属性和指定封装类型。

在使用字段时起别名（使用别名解决字段不匹配问题）

mybatis 的执行流程大概是：从数据库中出查询出数据，然后根据查询数据中的字段名找到封装类型中的同名属性，封装进去。如果从数据库中查询出来的字段名和封装类型中的属性对应不起来，就不能把这个字段的值封装进去。

解决方案：这个别名对应 pojo/bean 中的属性名就行了。

使用 sql 片段+ incliude 方式能一定程度上简化，但是还是麻烦。

参数传递(参数绑定)

MyBatis 传递参数是如何进行默认封装的？：

  • 单个参数：

    1. P0J0/bean类型：直接使用，属性名和參数占位符名称一致
    2. Map 集合：直接使用，键名和參数占位符名称一致
    3. CoLLection：封装为 Map 集合，可以使用@Param 注解，替换 Map 集合中默认的 arg 键名默认封装 2 个 key:value）
      • map.put（"arg0"，colLection集合;
      • map.put("collection"， collection集合);
    4. List：封装为 Map 集合，可以使用@Param 注解，替换 Map 集合中默认的 arg 键名（默认封装 3 个 key:value）
      • map.put（"arg0"，List集合）；
      • map.put("collection"，List集合);
      • map.put("List",List 集合);
    5. Array：封装为 Map 集合，可以使用@Param 注解，替换 Map 集合中默认的 arg 健名(默认封装 2 个 key:value）
      • map.put("arg0"，数组);
      • map.put("array"，数组);
    6. 其他类型(简单类型+字符串类型)：直接使用

  • 多个参数：封装为 Map 集合，可以使用 @Param注解，替换 Map 集合中默认的 arg 键名

  • xx xxxx(String username, String password) (不加注解这两个参数都使用了默认名字当 key)

    • map.put（"arg0"，参数值1）

    • map.put（"param1"，參数俏1）

    • map.put("param2"，参数值2）

    • map.put（"agr1"，参数值2）

  • xx xxxx(@Param("username") String username, String password)（第一参数加了注解使用注解中的名当key）

    • map.put（"username"，参数1）

    • map.put（"param1"，參数1）

    • map.put（"arg1"，参数值 2）

    • map.put（"param2"，參数值2）

为了在 sql 参数占位符 中使用更见名知意的标识符，推荐在编写接口中参数时，使用 @Para()注解。

使用参数

单个参数
  1. 参数占位符：
    1. #{}：执行 SQL 时，会将 #{} 占位符替换为 ?，将来自动设置参数值（类比 PrepraStatement 预编译）
    2. ${}：单纯拼接 SQL，会存在 SQL 注入问题
    3. 使用时机：
      • 参数传递，都使用 #{}
  2. parameterType：
    • 用于设置参数类型，该参数可以省略
  3. SQL 语句中特殊字符处理：
    • 转义字符(例如：小于号 &lt)
    • <![CDATA[内容]]> (在内容区的内容会当作纯文本，避免 <小于号 在 xml 中当成标签的开始)

    InputStream inputStream = Resources.getResourceAsStream(resource);  
    SqlSessionFactory sqlSessionFactory = new SqlSessionFactoryBuilder().build(inputStream);  
  
    /*获取sqlSession对象，用这个执行sql*/  
    SqlSession sqlSession = sqlSessionFactory.openSession();  
  
    //获取mapper对象  
    BrandMapper mapper = sqlSession.getMapper(BrandMapper.class);  
  
    //执行操作  
    int affectRow = mapper.deleteByIds(ids);  
    System.out.println(affectRow);  
    //提交事物（如果创建sqlSession时没有在openSession中填写为true则默认为不自动提交）  
    sqlSession.commit();  
  
    //释放资源  
    sqlSession.close();  
}
